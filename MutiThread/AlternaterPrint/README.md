# 多线程交替打印 ABC

## 问题

有三个线程，
- 线程A循环打印`A`
- 线程B循环打印`B`
- 线程C循环打印`C`
要求控制台输出:


```
txt ABCABCABCABC...
```

## 核心思路

多个线程需要同步执行顺序，否则线程调度顺序不可控。
可能输出：
    AAABBBCCC 
    ABCABCAB
    AACCBBB
等乱序结果。

---

## 解决方案
使用
- mutxt
- condition_variable
实现线程同步

---

## 核心状态变量

char g_ch = 'A'
用于表示哪个线程执行

## 线程A逻辑
等待 g_ch == 'A'
↓
打印A
↓
修改g_ch = 'B'
↓
通知其他线程

## condition_variable的作用
g_cv.wait(...)用于线程睡眠等待条件成立
避免 while(true) { 判断条件 } 这种CPU空转问题

## 为什么使用mutex
多个线程同时访问g_ch,因此必须保证共享数据线程安全，否则发生数据竞争属于未定义行为

## notify_one问题
对应两个线程，使用notify_one() 没有问题，但是三个及以上线程可能唤醒错误线程。

例如： 当前 g_ch = 'B' 理论上应该唤醒线程B，但notify_one()可能唤醒线程C
线程C发现条件不满足，会继续睡眠，最终所有线程进入等待状态，程序卡死。

## 正确做法 使用 notify_all
唤醒所有线程，只有条件满足的线程才能继续执行，其他线程会重新进入等待。
